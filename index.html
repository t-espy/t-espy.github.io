LEAN Research-to-Live Platform & lean-optimizer

Private QuantConnect LEAN research-to-live infrastructure platform with a public optimizer-engine subset. The public repository compiles C# strategies once, runs warm Docker containers with persistent .NET harness processes, and searches parameter space using GA, LHS, Bayesian/Optuna TPE, and local-grid stages. The private system adds IBKR data ingestion, WFO/WFAC credibility validation, portfolio construction, prediction bands, parameter-staleness monitoring, live/paper deployment packaging, dashboard approval, and post-trade analytics.

The platform has evaluated 7 strategy families across 257 symbols with WFAC verdicts, run 5,563 test plans and 36,254 WFO windows, and maintains 2,168 private test functions across 165 files. Only the Hull MA and Compression Breakout families passed credibility gates and were promoted to live/paper deployment; rotation and mean-reversion candidates were rejected by the validation framework.

A service-based Research Orchestrator runs unattended research funnels: data updates, parameter sweeps, WFO bake-offs, WFO-to-WFAC promotion, composition, and packaged backtests. A bounded LLM-guided Research Steward can read results and queue the next experiments through a restricted CLI. Live trading remains separate and manually approved.

langgraph-factory — Multi-Agent Code Generation with Local LLMs

Multi-agent code generation pipeline using LangGraph and locally-hosted LLMs via Docker Model Runner. A foreman model handles architecture planning and code review; a coder model handles file generation and fixes. Automated build-fix-regenerate loops recover from failures without human intervention. Runs entirely on local hardware with no external API dependency.

This project is part of the same AI-assisted engineering methodology used in the LEAN platform: explicit plans, bounded tool access, test discipline, durable decision records, and human-owned architecture rather than unstructured code generation.
